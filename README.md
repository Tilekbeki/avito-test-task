# Тестовое Авито приложение с объявлениями с AI фичей

## Технологический стек

- **React** v19.2.4
- **TypeScript** ~5.9.3
- **React Router** v7.13.2
- **Redux Toolkit** v2.11.2
- **React Query (TanStack Query)** v5.96.0
- **Ant Design** v6.3.5
- **Tailwind CSS** v4.2.2
- **SCSS Modules** для стилизации компонентов
- **Vite** v8.0.1
- **ESLint** + **Prettier** для линтинга и форматирования
- **Docker** + **Docker Compose** для контейнеризации
- **Vitest** + **Testing Library** для тестирования

## Функциональные возможности

### 1. Страница списка объявлений (`/ads`)
- Отображение всех объявлений продавца
- Шапка с заголовком «Мои объявления» и общим количеством
- Поиск по названию объявления
- Сортировка по новизне и названию
- Переключение вида (сетка/список)
- Пагинация по 10 объявлений на страницу
- Фильтрация по категориям и статусу «Требует доработок»
- Бейдж «Требует доработок» для незаполненных объявлений

### 2. Страница просмотра объявления (`/ads/:id`)
- Полная информация об объявлении в режиме просмотра
- Отображение характеристик товара в зависимости от категории
- Блок «Требуются доработки» с перечнем незаполненных полей
- Кнопка «Редактировать» для перехода к редактированию
- Навигация к списку объявлений

### 3. Страница редактирования объявления (`/ads/:id/edit`)
- Форма редактирования с валидацией полей
- Динамические поля характеристик в зависимости от категории
- AI-ассистент для улучшения описаний:
  - Кнопка «Улучшить описание» — генерирует продающий текст
  - Кнопка «Узнать рыночную цену» — предлагает актуальную цену
- Сохранение черновика в localStorage
- Автоматическое применение AI-рекомендаций
- Кнопки «Сохранить» и «Отменить»

### AI-интеграция
- Интеграция с Ollama (локальный запуск) через REST API
- Модель `llama3` для генерации описаний и рекомендаций цен
- Промпты оптимизированы под русскоязычные объявления
- Обработка контекста категории и характеристик товара

### Вопросы по архитектуре приложения

##### 1. Почему бэк не дает id в массиве объявлений? Пошел на бэк добавил в return дополнительное поле id

**Простыми словами:**

Изначально сервер возвращал список объявлений, но внутри каждого объявления не было поля `id`. Это было неудобно, потому что:

- **Фронтенду нужно знать id каждого объявления**, чтобы работать с ним (редактировать, открывать страницу объявления)
- Без явного id пришлось бы вытаскивать его из URL или придумывать костыли, что ненадежно и сложно

**Почему добавил на бэке:**

- **Сервер знает все про данные** - он создает эти объявления и точно знает их id
- **Фронт не должен гадать** - лучше сразу получить готовые данные, чем потом их вычислять
- **Так проще и надежнее** - одна правда, меньше ошибок

---

##### 2. Какая фильтрация: Серверная или Клиентская?

**Простыми словами:**

Выбрал **серверную фильтрацию** (фильтрую на бэке, а не на фронте).

**Почему:**

- **Представьте 10 000 объявлений** - если фильтровать на фронте, пришлось бы скачать все 10 000, а это долго и много трафика. Сервер отфильтрует и пришлет только нужные 20 штук

- **Пагинация (страницы)** - если фильтровать только то, что уже загружено на странице, то объявления с других страниц не попадут в результат. Сервер же видит все объявления сразу

- **Скорость работы** - сервер может быстро найти нужные объявления (особенно если в базе данных есть индексы), а фронту не придется перебирать тысячи объявлений в браузере

- **Единые результаты** - у всех пользователей фильтрация будет работать одинаково, потому что правила одни и те же на сервере

**Почему сначала думал иначе:**

Поначалу хотел сделать на клиенте (проще в разработке), но в процессе понял, что серверная фильтрация - это правильно для реального приложения.

---

##### 3. Будет ли пользователь делиться ссылкой на отсортированные объявления?

**Простыми словами:**

**Да, будет.** И это важная фишка приложения.

**Какие ситуации бывают:**

- Пользователь нашел квартиры до 5 миллионов рублей и хочет **скинуть ссылку другу** - "смотри, что я нашел"
- Человек отфильтровал ноутбуки по цене и **сохранил в закладки браузера**, чтобы вернуться завтра
- Менеджер настроил фильтр "активные объявления" и **отправил коллеге** для проверки
- Пользователь поделился в чате ссылкой на "новые объявления за сегодня"

**Что я сделал:**

- Привязал фильтры к **Search Params** (параметры в URL)
- Теперь URL выглядит так: `site.ru/ads?category=phones&price_from=1000&sort=price`
- Можно просто скопировать ссылку из браузера и отправить кому угодно

**Почему это правильно:**

- Пользователи привыкли к такому поведению (как на Авито, YouTube, картах)
- Не нужно придумывать свой шаринг - работает стандартный механизм браузера
- Можно вернуться к фильтру через неделю, просто открыв сохраненную ссылку

**Что если пользователь не будет делиться?**

Даже если не будет делиться, всё равно удобно:
- Обновить страницу - фильтры сохранятся
- Вернуться назад через кнопку браузера - фильтры те же
- Открыть в новой вкладке - всё как было

**Вывод:** Даже если 90% пользователей не делятся, оставшимся 10% эта функция очень важна. А для всех остальных это просто удобный интерфейс.

##### 4. Почему выбрал React Query?

**Коротко и просто:**

React Query кеширует данные с сервера и сам решает, когда их обновлять.

**Что дает:**

- **Пользователь ушел со страницы и вернулся** → данные показываются мгновенно из кеша, не нужно грузить заново

- **Данные устарели** → React Query в фоне сам сходит на сервер и обновит их

- **Пользователь добавил новое объявление** → React Query сам перезапросит список, не нужно писать дополнительный код

- **Не нужно возиться с загрузкой и ошибками** → библиотека сама дает флаги `isLoading` и `isError`

## Запуск проекта

### Вариант 1: Локальный запуск (без Docker)

#### Предварительные требования
- Node.js v20+
- npm или pnpm

#### Шаги для запуска

1. **Установите Ollama** (для AI-функций)
   ```bash
   # Скачайте и установите Ollama с официального сайта
   # Запустите Ollama сервер
   ollama serve
   ```

2. **Загрузите модель llama3**
   ```bash
   ollama pull llama3
   ```

3. **Запустите серверную часть**
   ```bash
   cd server
   npm install
   npm start
   ```
   Сервер будет доступен по адресу: http://localhost:8080

4. **Запустите клиентскую часть**
   ```bash
   cd front
   npm install
   npm run dev
   ```
   Приложение будет доступно по адресу: http://localhost:3000

##### Вариант 2: Запуск через Docker 
```bash
 docker compose up -build 
```

##### Загрузить модель
``` bash 
docker exec -it avito-test-task-ollama-1 ollama pull llama3
```
